DESIGN PRINCIPLES
────────────────────────────────────────────────────────────────────────────
✅ Single Responsibility - One tool per task
✅ Composition over Inheritance - Independent functions
✅ Type Safety - Full TypeScript, no `any` types
✅ Separation of Concerns - Prompts, utils, tools isolated
✅ DRY - Reusable data formatters and utilities
✅ SOLID Principles - Extensible, testable design
✅ Error Handling - Consistent ToolResponse wrapper
✅ Minimal Dependencies - Only Anthropic SDK + dotenv

CONSISTENT TOOL PATTERN
────────────────────────────────────────────────────────────────────────────
Input → Validate → Format → Call Claude → Parse → Return ToolResponse

Each tool returns:
{
  success: boolean,
  data?: T,
  error?: string,
  reasoning?: string,
  timestamp: Date
}
